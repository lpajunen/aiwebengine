How It Works

This demo shows how clients specify message filtering criteria when subscribing to selective broadcasting:

  1. Connection Metadata: When connecting to a stream, include query parameters that define your metadata
  2. Server-Side Filtering: The server uses this metadata to filter which connections receive messages
  3. Selective Broadcasting: Use sendStreamMessageToConnections() with filter criteria

Example URLs:

  • /test/chat?user_id=user123&room=general - Connects with user and room metadata
  • /test/metadata-demo?demo=true&user_id=user123 - Connects with demo flag

Filter Examples:

  • {user_id: "user123", room: "general"} - Send to specific user in specific room
  • {demo: "true"} - Send to demo connections only
  • {} - Send to ALL connections (empty filter)
